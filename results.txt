The poeninja_snapshot service is failing due to two interconnected issues:

**Step 1 Issue (0 rows from PoeNinja API):**
The service calls `ml_workflows.snapshot_poeninja()` which uses `PoeNinjaClient.fetch_currency_overview(league)`. Based on the logs showing "Snapshot complete: 0 rows", the PoeNinja API is returning empty data. Testing the API directly with `curl -s "https://poe.ninja/api/data/currencyoverview?league=Mirage&type=Currency"` returned empty response, indicating either:
1. The PoeNinja API is temporarily unavailable or rate-limiting
2. The league parameter "Mirage" is incorrect for current PoeNinja data
3. Network connectivity issues preventing API access

**Step 3 Issue (ClickHouse timeout):**
When Step 1 returns 0 rows, the `build_fx` function still inserts a fallback row (chaos=1.0), but the `normalize_prices` function then executes a complex JOIN query against `v_ps_items_enriched` view that times out. This suggests:
1. The ClickHouse query in `normalize_prices` is too complex or lacks proper indexing
2. The `v_ps_items_enriched` view may be referencing empty or poorly performing base tables
3. The timeout setting (30 seconds from ClickHouseClient) may be insufficient for the query complexity

**Root Cause Analysis:**
The service is designed to handle empty PoeNinja responses gracefully (via fallback FX rates), but the downstream normalization query fails when there's no recent item data to process. This creates a cascading failure where:
1. No PoeNinja data → 0 rows in raw_poeninja_currency_overview
2. Fallback FX data inserted (1 row for chaos currency)
3. Normalization query attempts to join empty/no item data → times out

**Immediate Fixes Needed:**
1. Add better error handling for empty PoeNinja responses to short-circuit the pipeline
2. Increase ClickHouse timeout for complex analytical queries
3. Validate that the PoeNinja API is accessible and returning expected data format
4. Check if the `v_ps_items_enriched` view has recent data to process
